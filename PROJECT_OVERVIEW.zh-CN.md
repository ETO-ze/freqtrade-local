# 项目概览

## 这个项目在做什么

这个项目把本地模型研究和服务器侧 Freqtrade 执行连接成了一条完整链路。

- 本地机器：行情刷新、动态币池生成、筛选、模型训练、候选回测、审批。
- 服务器：Bot 执行、受保护的 UI、HTTPS 访问，以及公开只读看板数据。

## 核心组件

### OpenClaw 研究闭环
- 刷新 OKX 永续行情数据
- 基于流动性和稳定性代理指标构建动态山寨币池
- 运行 robust screen
- 训练树模型
- 生成 `tradable / observe / pause` 决策
- 对候选配置做回测
- 只有通过门槛才允许 promotion

### Freqtrade 执行层
- 山寨执行线
- 主流执行线
- HTTPS 保护的 Web UI
- 提供给客户端的 API

## 当前设计取舍

### 动态币池
- `stable` 使用更窄的动态币池（`top_n = 15`）
- `fast` 使用更宽的动态币池（`top_n = 20`）

这样拆分的原因是：
- 正式链路优先保证质量
- 轻量链路优先保证覆盖和早期观察

### 审批门槛
Promotion 门槛保持严格。

- 收益必须足够高
- 回撤必须受控
- 交易数必须足够
- 允许高收益绕过交易数，但仍然偏保守

## 当前约束

现在流程结构已经跑通，但在刷新到 4 月新数据后，当前结果仍然没有通过 stable promotion gate。

这意味着：
- 自动化链路本身是正常的
- 服务器运行时没有被弱候选覆盖
- 下一步需要优化的是因子质量，而不是流程 plumbing

## 下一步优化重点

1. 继续削弱 `mark_premium` 家族在模型选择中的统治力
2. 提高正交因子的实际贡献
3. 给动态扩池增加更严格的噪音过滤
4. 保持 promotion 严格，只放宽研究宇宙，不放宽上线标准
